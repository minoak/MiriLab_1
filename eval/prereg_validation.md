> 사전 지식 공개: Gemini 질감 프로브(48콜)에서 10억 정책에 stance 3분화·불만 분화를
> 이미 관찰했다. 아래 예측은 그 관찰 이후에 세운 것이다(완전 블라인드 아님 — 정직).

## ② 타당성 — 그라운딩 ON/OFF ablation

**질문**: 결과 차이를 만드는 것이 페르소나인가? (LLM 변덕이 아니라)

**설계**: 정책 = 청년 월세 지원(샘플 원문 고정). 24명 × {ON, OFF} × 3회 반복 = **144콜**.
- ON: 인물 카드 주입 (`build_react_messages(grounded=True)`)
- OFF: 카드 없는 익명 시민 (`grounded=False`) — 무대·형식 동일, 카드만 제거(단일 변인)
- 반복 3회: 같은 입력의 회차 간 출렁임(노이즈 바닥)을 같이 잰다

**사전등록 판정 기준** (주축 = benefit·intent, 0~100 변환 후):

| # | 지표 | 기준 | 묻는 것 |
|---|---|---|---|
| V1 | 대상 분리: 대상자(is_target) vs 비대상자의 benefit 평균 차 | ON에서 ≥ 30점 **그리고** OFF에서 < 15점 | 점수가 실제 자격조건을 따라가는가 |
| V2 | 수렴타당도: policy_fit ↔ benefit Spearman r | ON에서 r ≥ 0.5 **그리고** ON−OFF 차 ≥ 0.3 | 결정론 적합도와 LLM 점수의 정렬 |
| V3 | 분산비: 페르소나 간 std (ON) / (OFF) | benefit 축에서 ≥ 1.5 | ON이 갈리고 OFF가 뭉개지는가 |
| V4 | 노이즈 바닥: 페르소나 간 std / 회차 간 std (ON) | ≥ 2.0 | 차이가 LLM 변덕보다 충분히 큰가 |

- V1·V2 = 주판정(이게 핵심 주장). V3·V4 = 보조(공격 방어).
- OFF 조건은 전원 같은 프롬프트 → 페르소나 간 차이는 순수 샘플링 노이즈여야 함.
- is_target / policy_fit 은 결정론(LLM 0) — 비교 기준이 LLM 과 독립.

## ③ 견고성 — 극단 정책 테스트

**질문**: 말도 안 되는 입력에도 상식 방향으로 반응하는가?

**설계**: 24명 × {천장, 바닥} × 3회 반복 = **144콜**. 정책 원문(고정):

- **천장**: "정부는 전 국민에게 1인당 10억 원을 일시금으로 지급한다.
  신청 절차 없이 전 국민의 계좌로 자동 입금되며, 소득·연령·재산 조건은 없다."
- **바닥**: "정부는 전 국민에게 1인당 1원을 일시금으로 지급한다.
  신청 절차 없이 전 국민의 계좌로 자동 입금되며, 소득·연령·재산 조건은 없다."

(형식·문장 구조를 동일하게 — 금액만 다른 쌍둥이 정책. 다른 변인 차단.)

**사전등록 판정 기준** (3회 평균 기준, 회차별 일관성도 보고):

| # | 지표 | 기준 | 상식 |
|---|---|---|---|
| R1 | 천장: 대상 자가인식(eligibility=target) 비율 | ≥ 80% | 전 국민 지급이니 거의 전원 "내가 대상" |
| R2 | 천장: 수혜 인식(benefit ≥ 75) 비율 | ≥ 50% | 진짜 주면 혜택이긴 함 (실현 의심으로 일부 유보 허용) |
| R3 | 천장: 사회혼란도(불만 평균) | 바닥(1원)보다 높음 **그리고** ≥ 20 | 좋아도 불안·의심 동반 (공짜 10억이 평온하면 비정상) |
| R4 | 바닥: 무관·무효용(benefit=50 또는 eligibility=not_target 무관 응답) 비율 | benefit≤50 비율 ≥ 70% | 1원은 아무 의미 없음 |
| R5 | 바닥: 적극 신청 의향(intent > 50) 비율 | ≤ 20% | 1원 받자고 움직이지 않음 |
| R6 | 순서: 천장 intent_pos > 바닥 intent_pos | 부등호 성립 | 금액 크기에 의향이 반응 |

- 6항 중 **5항 이상 = 견고성 통과**, 4항 이하 = 시스템 점검 (기준 다듬기 금지 — 시스템을 고친다).
- 알려진 측정 한계(사전 공개): '실현 기대' 축이 없어 "안 믿지만 받으면 좋지"가
  benefit/intent 에 섞여 들어옴(프로브1 진단). R2·R3 임계는 그걸 감안해 보수적으로 잡음.

## 공통

- 비용 추정: 288콜, 입력 ~3k tok/콜 → 총 ~$1 안팎 (Gemini flash).

  현 프롬프트 레이아웃의 캐시 적중률 실측(발표 후 prefix 최적화의 근거 데이터).
- 산출: `eval/ablation_results.json` + `eval/robustness_results.json` + 각 md/png.
- 구버전 eval JSON(cluster/ablation, 구프롬프트·구척도·gpt-4o-mini)과 분포 비교 금지.
- 판정이 나쁘면: 기준·임계값을 다듬지 않는다. 시스템을 고치고 **전체 재실행**한다.
