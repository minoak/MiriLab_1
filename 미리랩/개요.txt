### 0. 개요

### 1. 문제인식

- 프로젝트 목표 및 목적  구현하고자 하는 아이템의 목표 및 목적

정부와 지자체는 매년 수천 건의 복지·지원 정책을 발표하지만, 정작 필요한 시민에게 전달되지 못하는 경우가 많다. 특히 다음과 같은 구조적 문제가 존재한다.

- 정보 격차 시민이 자신이 정책 대상자인지 인지하지 못함
- 이해도 부족 정책 문구가 어렵고 신청 조건이 불명확함
- 디지털 장벽 시니어·정보 취약계층의 온라인 신청 어려움
- 확산 부재 정책이 발표되어도 필요한 사람에게 도달하지 못함

본 프로젝트의 목표는 정책이 실제 시민에게 배포되기 전, AI 가상 시민 사회에서 먼저 실험해 정책 수용도·이해도·신청 장벽을 사전에 분석하고, 개선안을 제시하는 시스템을 구현하는 것이다.

핵심 목적 3가지

1. 정책 문서를 RAG로 분석하여 핵심 정보를 자동 추출
2. 다양한 시민 페르소나 Agent가 정책에 반응하는 과정을 시뮬레이션
3. 시뮬레이션 결과를 바탕으로 정책 개선안을 자동 생성
- 아이템의 독창성 

 유사 서비스  주요 기능  한계 
 ---  ---  --- 
 정부 24 챗봇  정책 Q&A 응답  단순 검색, 사용자 맞춤 없음 
 복지로  복지 서비스 검색·추천  수혜 조건 매칭만 가능 
 일반 RAG 챗봇  문서 검색 후 답변  사회적 파급 효과 분석 없음 
 Agent 시뮬레이션 연구  학술 연구 수준  실사용 서비스 미존재 
- 차별성

[기존 RAG 서비스]
정책 문서 → 검색 → 단일 답변

[본 프로젝트]
정책 문서 → RAG 검색 → 시민별 판단 → 감성 분석 → 네트워크 전파 → 사회 반응 시각화 → 개선안 제안

- 가상 시민 Agent 시스템 단순 답변이 아닌 20~30명의 개성 있는 시민 Agent가 각자 다르게 반응
- 실시간 SNS형 채팅 시뮬레이션 카카오톡SNS 스타일의 채팅방에서 시민 간 정보 전파 과정을 실시간으로 시각화
- 사회 혼란도 대시보드 가상 시민의 댓글·반응 감성을 점수화하여 `사회 혼란도 온도계` 등 직관적인 그래프로 표현
- 정책 문의 게시판 자동 답변 사용자 또는 가상 테스터가 게시판에 댓글을 남기면 AI가 자동으로 맞춤 답변 제공
- 직접 개입 AB 테스트 정책 문구나 신청 방식을 변경했을 때 이전·이후 시민 반응 변화를 비교 분석
- 아이디어와 유사한 프로젝트 유무를 확인
- 구현하려는 아이템과의 차별성을 기재

### 2. 개발 및 연구 내용

- 구현 내용 상세
    
    ```
    [사용자]
        │
        ▼
    [정책 입력 화면] ──→ 샘플 정책 선택 or 직접 입력
        │
        ▼
    [RAG 엔진]
     ┌──────────────────────────────────────────┐
     │  정책 문서 chunking → embedding → 검색       │ 
     │  대상자  혜택  신청 방법  기간 추출           │
     └──────────────────────────────────────────┘
        │
        ▼
    [시민 Agent 시스템]
     ┌──────────────────────────────────────────┐
     │  페르소나 20~30명 생성                       │
     │  각 시민 나이, 소득, 디지털 리터러시,           │
     │          정부 신뢰도, 사회 연결망 보유          │
     │  ↓                                       │
     │  이해도  수혜 가능성  신청 의향              │
     │  불만도  공유 가능성 점수 산정                 │
     └──────────────────────────────────────────┘
        │
        ├──→ [SNS형 실시간 채팅방] 시민 간 정보 전파
        │
        ├──→ [감성 대시보드] 사회 혼란도  수용도 온도계
        │
        ├──→ [네트워크 전파 그래프] 정보 확산 경로 시각화
        │
        ├──→ [정책 게시판] 자동 댓글 답변 시스템
        │
        ├──→ [AB 테스트] 정책 개입 전후 비교
        │
        └──→ [개선 제안] 쉬운글 변환 + 정책 개선안 출력
    ```
    
- 구현체의 구체적인 구조를 설명
    - 요구되는 데이터, 관련 연구 문헌 혹은 웹페이지를 통한 개요 정리, 사진 첨부도 가능, 문헌 인용은 레퍼런스에 기재
- 개발 아이템 기대효과
    - 서비스로 배포되었을 때의 활용 용도 등의 가능성을 자유롭게 정리

### 3. 실행 계획

- 프로젝트 완성을 위한 전략
    - 기간 내 목표를 완성하기 위한 전략 구상 - 일정, Task 분배 등 고려
    - 데이터 선정 및 EDA 등 단계별 소요기간 및 프로젝트 발표회의 PT 준비까지의 전체 일정을 위한 Task를 기획

        팀을 구성하는 인원은 최소 3명, 최대 5명으로 필요인원만큼 기재

## 2. 개발 및 연구 내용

### 핵심 모듈별 설명

① RAG 엔진

- 정책 문서를 청크 단위로 분할, ChromaDB 또는 FAISS에 임베딩 저장
- 쿼리 시 유사도 기반 검색으로 대상자, 혜택, 신청 조건, 주의사항 자동 추출
- 정책 키워드를 활용한 맞춤 조건 매칭 (본인 조건 입력 시 수혜 여부 추천)

② 가상 시민 Agent

```python
# 시민 페르소나 예시
{
  name 김영자,
  age 76,
  type 독거 시니어,
  income_level low,
  digital_literacy 0.15,
  government_trust 0.5,
  social_network [자녀, 복지관, 경로당]
}
```

- 각 시민은 RAG 결과를 입력받아 LLM 기반으로 반응 생성
- 이해도, 수혜 가능성, 신청 의향, 불만도, 공유 가능성을 0~100점으로 점수화
- 예상 행동 (신청 시도  포기  가족 공유  주민센터 문의 등) 생성

③ SNS형 실시간 채팅방

- 카카오톡SNS 스타일의 UI로 시민 간 정책 정보 전달 과정을 라이브로 시뮬레이션
- 시민이 다른 시민에게 정보를 공유하거나 오해가 퍼지는 과정을 채팅 메시지 형태로 표현
- 전파 턴(Turn) 구조`1턴 복지관 직원 확인 → 2턴 시니어 5명 안내 → 3턴 자녀단톡방 공유 → 4턴 지역 커뮤니티 확산`

④ 감성 대시보드

- 시민 댓글·반응 텍스트를 감성 분석 (긍정부정혼란)
- `사회 혼란도`, `정책 수용도`, `신청 의향 지수` 등을 실시간 온도계·게이지 그래프로 시각화
- 시간 흐름에 따른 감성 변화 추이 차트

⑤ 정책 문의 게시판 자동 답변

- 사용자 또는 가상 테스터가 게시판에 질문댓글을 남기면
- RAG 엔진이 정책 문서에서 답변을 검색·생성하여 자동 댓글로 달아줌
- 가상 시민도 게시판에 반응을 남겨 현실감 있는 시뮬레이션 구현

⑥ AB 테스트 (직접 개입 기능)

- 관리자가 정책 문구, 신청 방식, 안내 언어(쉬운글) 등을 직접 수정
- 수정 전후 시민 반응 변화를 나란히 비교 (이전 혼란도 vs 이후 혼란도)
- 어떤 개선이 실제로 효과 있는지 데이터로 검증

⑦ 네트워크 전파 그래프

- 복지관 → 시니어 → 자녀 → 가족 단톡방 → 지역 커뮤니티로 이어지는 정보 확산 경로를 네트워크 그래프로 시각화
- 고립된 시민(정보 미도달), 오해 발생 지점, 신청 포기 지점을 노드 색상으로 구분

### 개발 아이템 기대효과

정책 입안 단계에서의 활용

- 정책 발표 전 시뮬레이션으로 이해도 낮은 집단, 신청 장벽이 높은 집단을 사전 파악
- 쉬운글 변환·신청 절차 개선 등 구체적 개선안을 데이터 기반으로 도출

복지 현장에서의 활용

- 복지관·주민센터 담당자가 어떤 시민 유형이 도움 필요한지 우선순위 파악
- 정보 확산 채널 설계 (경로당, 가족 단톡방 활용 전략)

서비스 배포 시 활용 가능성

- 중앙부처·지자체 정책 시뮬레이션 도구로 도입 가능
- 시민 단체·NGO의 정책 모니터링 플랫폼으로 확장
- 정책 리터러시 교육 콘텐츠로 활용

## 3. 실행 계획

### 프로젝트 완성을 위한 전략

전략 원칙

1. MVP 우선 4일 내 핵심 기능(RAG + Agent 반응 카드 + 감성 대시보드) 완성 후 추가 기능 확장
2. 병렬 작업 RAG 파이프라인, Agent 설계, UI 개발을 동시에 진행
3. 데이터 고정 샘플 정책 2~3개를 미리 선정하여 데이터 수집 시간 최소화
4. 모듈화 각 기능을 독립 모듈로 개발 → 최종 Streamlit에서 통합

### 팀장 및 팀원의 역할 분배

 역할  주요 담당 업무  세부 Task 
 ---  ---  --- 
 팀장 (A)  전체 조율 + RAG 파이프라인  정책 문서 수집·전처리, chunking, embedding, ChromaDB 구축, 검색 로직 구현, 최종 통합

### 프로젝트 아이템 개요 (요약)

### 아이템 소개

AI 시민 사회 정책 실험실은 정부·지자체 정책 문서를 RAG로 분석하고, 20~30명의 가상 시민 Agent가 각자의 상황(나이·소득·디지털 리터러시 등)에 따라 정책에 어떻게 반응하는지 시뮬레이션하는 플랫폼이다.

핵심 기능

- 정책 문서 RAG 검색 및 핵심 정보 자동 추출
- 가상 시민 Agent 반응 생성 (이해도, 수혜 가능성, 신청 의향, 불만도 점수화)
- SNS형 실시간 채팅방 시뮬레이션 (카카오톡 스타일 UI)
- 감성 대시보드 (사회 혼란도·정책 수용도 온도계 그래프)
- 정책 문의 게시판 자동 답변 시스템
- 정보 전파 네트워크 그래프 시각화
- 직접 개입 AB 테스트 (정책 수정 전후 비교)
- 쉬운글 변환 및 정책 개선 제안 자동 출력

### 아이템의 특징 및 차별성

 구분  기존 서비스  본 프로젝트 
 ---  ---  --- 
 접근 방식  질문 → 검색 → 단일 답변  정책 → 다양한 시민 반응 → 사회적 파급 분석 
 사용자 관점  정책을 찾는 사람  정책을 만드는 사람 (입안자·연구자) 
 시각화  텍스트 답변 위주  SNS 채팅방, 온도계, 네트워크 그래프 
 피드백 루프  없음  AB 테스트로 개선 효과 즉시 확인 
 감성 분석  없음  사회 혼란도·수용도 실시간 수치화 
 확산 시뮬레이션  없음  가족·복지관·커뮤니티 정보 전파 경로 추적

### 시스템 구성 개요도

```
┌─────────────────────────────────────────────────────────┐
│                 AI 시민 사회 정책 실험실                  │
├─────────┬──────────┬──────────┬──────────┬──────────────┤
│ Tab 1   │  Tab 2   │  Tab 3   │  Tab 4   │   Tab 5      │
│정책 입력 │RAG 검색  │시민 반응 │네트워크  │ 개선 제안    │
│         │ 결과     │ 대시보드 │ 전파도   │              │
├─────────┴──────────┴──────────┴──────────┴──────────────┤
│  SNS 채팅방 │ 게시판 자동답변 │ AB 테스트 비교         │
└─────────────────────────────────────────────────────────┘

핵심 백엔드
[정책 문서] → [RAG 엔진] → [시민 Agent x 20~30명]
                               ↓          ↓          ↓
                         [감성 분석] [전파 모델] [개선 제안]
                               ↓          ↓          ↓
                         [대시보드]  [그래프]   [쉬운글]
```

---

 이 프로젝트는 단순히 정책을 설명하는 서비스가 아니다.
 
 
 정책을 시민 입장에서 미리 실험해보고, 누가 이해하지 못하는지, 누가 신청하지 못하는지, 어디서 오해가 생기는지를 보여주는 가상사회 시뮬레이션 서비스다.
